# Contributing

Thanks for helping improve Grok Build for Pi.

1. Fork the repository and create a focused branch.
2. Install the pinned development dependencies with `npm ci`.
3. Add or update a regression test for behavior changes.
4. Run `npm run check` before opening a pull request.
5. Explain any intentional difference from the upstream Claude Code plugin.

Never commit credentials, Pi/Grok session files, repository transcripts, generated state, or logs. Security issues should follow [SECURITY.md](SECURITY.md) instead of a public issue.
